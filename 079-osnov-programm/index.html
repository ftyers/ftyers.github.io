Description

A gentle introduction to corpus processing and programming for computational linguists. The course is structured in two parts, the first five classes are familiarisation with programming in Python and the UNIX shell, the remaining three are implementing a program of your choice to solve a personal need related to linguistics or natural-language processing. Every week we will do a combination of short lectures where I will introduce some concepts, and then the remainder of the course will be supervised practical work — that is, programming.

Course objectives

By the end of this course you should be:
  • familiar with the syntax of the Python programming language
  • able to navigate the filesystem using the UNIX shell
  • capable of efficiently using online resources to guide your own learning
  • able to use Python to process corpora from the web
  • able to use Python to implement a simple NLP pipeline from scratch

Prior experience

Those of you who are already fairly confident in programming in Python may opt to not follow the lectures and simply work on the practicals. If you finish the practicals early, you can start work on your personal project (see below).

Extra help

If you are having problems keeping up with the workload and would like some one-on-one help, on the 18th and 19th, before the final class, you can sign up for an hour's help completing the practicals and homework tasks. If you'd like this, you can sign up using this Doodle.

Project work

Marking

  • 40% Project
  • 25% Practicals
  • 25% Homework
  • 10% Active participation

Project: The project will encompass a good proportion of the class time and homework for the last three classes. You should start thinking from the first class what you might be interested in working on. If you cannot come up with any ideas, then I will give a number of options, or come and talk to me. The project should be non-trivial and test and expand your knowledge in some way. It should contain an evaluation component, either for efficiency of implementation or in terms of accuracy for some task. One of the most important aspects of programming is learning to use the computer to scratch an itch `удовлетворить личное желание' the project will ensure you are able to do that.

Practicals: Most of the course will be made up of practical sessions. I will evaluate your progress after each session.

Homework: Homework that isn't just reading will be submitted through Github, and will need to be completed before the following lesson. Your Github repository should be called 2017-osnov-programm and have the following subdirectories: corpus for your (sub-)corpus from Wikipedia, project for your project work and homework for miscellaneous homework tasks. If you finish all practical work in a session, you can start on the homework.

Active participation: Beyond simply showing up, I encourage you to contribute to discussions by asking questions, answering questions, making relevant comments, helping classmates and asking for help with in-class activities, etc. There are no stupid questions — I want to make sure everyone grasps the concepts, and many are not as straightforward as they may first seem (or as I think they are). You are also expected to have read any assigned readings before class.

Project ideas

As the project will take up approximately 12 hours of work time (9 hours of class time and 3 hours of homework time), the topic of the project should be sufficiently in depth to fill the time and should be agreed with me before you start.
  • Do an experiment using data from the treebanks in Universal Dependencies (for example look at factors that influence word order)
  • Do an experiment using the World Atlas of Linguistic Structures (WALS)
  • Write a program to classify words in a particular language by declension class
  • Write a program to convert between two tagsets, e.g. mystem and Universal Dependencies
  • Improve the pipeline from your practical work to cover the the majority of cases in the Wikipedia corpus.
  • Write an orthography to IPA convertor for a language of your choice.
  • Make a cool visualisation of some data using a Python library.
  • Write a program to make a dataset (e.g. for keyword extraction, named-entity recognition) from Wikipedia
  • ...
